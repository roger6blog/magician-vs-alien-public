# 魔數結晶提案

0925 設計文件中的「星星」（經濟用途：購買裝備等）改名為「魔數結晶」，與[星數計算提案](star-calculation-proposal.md)中的星星（教育表現回饋）區分。

---

## 一、魔數結晶是什麼？

- **功能**：遊戲內的經濟貨幣，可用於購買裝備等
- **取得難度**：遊戲難度越高越不容易取得，這是合理的經濟平衡
- **劇情包裝**：時間到結算星星後，由數之長老帶出——「星球的能量穩定之後，產生了魔數結晶！」

---

## 二、計算方式

結合 0925 的兩個核心邏輯，同時移除「限制重玩兩次」的限制：

1. **愛心表現**（0925 精神：死亡越少，獎勵越高）
2. **遊玩時間**（0925 精神：越早放棄，獎勵越低）

### 基礎結晶（依死亡次數）

| 死亡次數 | 基礎結晶 |
|---------|---------|
| 0～1 次 | 5 顆 |
| 2～4 次 | 4 顆 |
| 5～8 次 | 3 顆 |
| 9～12 次 | 2 顆 |
| 13 次以上 | 1 顆 |

### 時間係數（依有效遊玩時間）

| 有效遊玩時間 | 係數 | 對應 0925 邏輯 |
|------------|------|--------------|
| 8 分鐘以上 | ×1.0 | 等同「選擇重玩並完成」，完整獎勵 |
| 6～8 分鐘 | ×0.7 | 等同 0925「剩餘 ≤ 1 分鐘」，約 60% 獎勵 |
| 4～6 分鐘 | ×0.4 | 等同 0925「剩餘 ≤ 2～3 分鐘」，約 40% 獎勵 |
| 不到 4 分鐘 | ×0 | 等同 0925「剩餘 > 3 分鐘」，歸零 |

### 最終結晶 = 基礎結晶 × 時間係數（無條件捨去）

展開成完整結晶表：

| | 玩 8 分鐘以上 | 玩 6～8 分鐘 | 玩 4～6 分鐘 | 不到 4 分鐘 |
|---|---|---|---|---|
| 0～1 次死亡 | **5** | 3 | 2 | 0 |
| 2～4 次死亡 | **4** | 2 | 1 | 0 |
| 5～8 次死亡 | **3** | 2 | 1 | 0 |
| 9～12 次死亡 | **2** | 1 | 0 | 0 |
| 13 次以上 | **1** | 0 | 0 | 0 |

---

## 三、為什麼需要時間係數？

在無限重玩的機制下，如果結晶只看死亡次數，孩子可能會發現「少死幾次就停下來不玩」能拿到更多結晶——這會鼓勵孩子**停止練習數學**，是最糟糕的結果。

加入時間係數後，越早停止遊玩、結晶扣得越重（甚至歸零），確保**繼續玩永遠比停下來好**：

| 情境 | 死亡 | 遊玩時間 | 結晶 |
|------|-----|---------|------|
| 死 2 次就不玩了 | 2 次 | 3 分鐘 | **0 顆** |
| 繼續玩到底 | 7 次 | 10 分鐘 | **3 顆** |
| | | | |
| 死 1 次，玩到一半停 | 1 次 | 5 分鐘 | **2 顆** |
| 繼續玩到底 | 4 次 | 10 分鐘 | **4 顆** |
| | | | |
| 表現好又玩滿 | 1 次 | 10 分鐘 | **5 顆** |

## 四、待討論事項

1. **結晶的死亡次數門檻是否合理？** 目前設定 0～1 次死亡 = 5 顆、2～4 次 = 4 顆等，需確認是否符合實際遊玩節奏
2. **時間係數的分段是否合理？** 目前不到 4 分鐘直接歸零，是否太嚴格？
3. **魔數結晶可以用來買什麼？** 需要設計結晶的消費用途（裝備、道具等）
